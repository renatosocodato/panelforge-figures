# Getting help / Support

Thanks for using **panelforge-figures**. Here's how to get help, by what you need.

## Usage questions ("how do I…?")

GitHub Discussions is **not enabled** for this project, so usage questions are
welcome as **GitHub issues** — open one and the maintainer will help:

- **[Ask a question / report a problem](https://github.com/renatosocodato/panelforge-figures/issues/new/choose)**

Before opening one, these usually answer the most common questions:

- [`README.md`](README.md) — install and the end-to-end workflow
- [`docs/quickstart.md`](docs/quickstart.md) — quick start
- [`docs/AGENT_RECIPES.md`](docs/AGENT_RECIPES.md) — the recipe-discovery tutorial
- `figures --help` (and `figures <command> --help`) — the full CLI surface
- [`docs/`](docs/) — recipe catalogs, manifest schema, themes, palettes, adapters

## Bugs and feature requests

Use the issue templates — they prompt for the details needed to act quickly:

- **[Bug report](https://github.com/renatosocodato/panelforge-figures/issues/new?template=bug_report.md)**
- **[Feature request](https://github.com/renatosocodato/panelforge-figures/issues/new?template=feature_request.md)**

## Contributing

See [`CONTRIBUTING.md`](CONTRIBUTING.md) for the local development loop (it
mirrors the CI gates exactly) and [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md) for
community expectations.

## Security or privacy issues

**Do not** open a public issue for a security or data-privacy problem. Report it
privately via a
[GitHub Security Advisory](https://github.com/renatosocodato/panelforge-figures/security/advisories/new)
(preferred) or by e-mail to the maintainer — see [`SECURITY.md`](SECURITY.md).

## Citing panelforge-figures

If you use it in published work, please cite the archived release — see the
**Citation** section of [`README.md`](README.md) and
[`CITATION.cff`](CITATION.cff).
